# Internationalization (i18n)

Disk Cleaner uses GNU gettext. Source code is in English; translations
live under [`po/`](../po/) as `.po` files.

## Quick start

### Add a new language

```bash
cd po
msginit -i messages.pot -l de_DE.UTF-8 -o de.po  # for German
# Translate inside de.po, then:
make compile                                       # produces de.mo
```

### Update an existing language

```bash
cd po
make pot          # refresh .pot from source (xgettext)
make update       # sync all .po files with .pot (msgmerge)
# fill in new fuzzy/empty msgstr entries inside the .po
make compile      # build .mo
```

### Just the numbers

```bash
cd po
make check        # per language: N translated, M fuzzy, K untranslated
```

## Developer workflow — when adding a new string

```python
# 1. Import
from disk_cleaner.i18n import _, ngettext

# 2. Wrap user-facing strings with _()
button = Gtk.Button(label=_("Smart scan"))

# 3. Use ngettext for plurals
msg = ngettext("{n} item", "{n} items", n).format(n=n)

# 4. Refresh the .pot (before committing)
cd po && make pot
```

**RULE:** every string the user can see must be wrapped in `_()`.
Exceptions:
- Module docstrings (developer)
- Inline comments (developer)
- Magic IDs / dict keys (`"sys"`, `"proj"`, signal names)
- File paths, URLs, regex patterns
- Internal log messages (stderr/file only)

## Runtime language selection (priority order)

1. `DISK_CLEANER_LANG=tr` env var (dev/test/CI override)
2. The `language` key in `settings.json` (user preference — from the Settings UI)
3. System `LANG` / `LC_MESSAGES` env (native gettext fallback)
4. English if none of the above (source language)

The language is changed from the Settings dropdown; **a restart is
required.** Runtime re-rendering is not supported (existing widgets are
not re-translated).

## CI guard

[`.github/workflows/i18n.yml`](../.github/workflows/i18n.yml) runs on every PR:

- Runs `make pot` → builds an up-to-date .pot
- Checks that every `.po` is in sync with the .pot
- Catches placeholder drift via `msgfmt --check-format`
- **Fails** if Turkish characters leak into the source code

Local equivalent:

```bash
cd po && make pot && make update && make compile && make check
grep -rnE '"[^"]*[şŞıİğĞçÇöÖüÜ][^"]*"' ../disk_cleaner/ --include='*.py' \
  | grep -v -E ':[[:space:]]*#|"""'
```

## Plural forms

| Language | Header | Notes |
|---|---|---|
| en | `nplurals=2; plural=(n != 1);` | 1 singular, rest plural |
| tr | `nplurals=2; plural=(n > 1);` | 0 and 1 are singular; Turkish nouns are not grammatically pluralized after a number, but gettext convention still requires two forms |

For new languages see: [Gettext manual — Plural Forms](https://www.gnu.org/software/gettext/manual/html_node/Plural-forms.html)

## String freeze policy

A **string freeze** is applied during release preparation:

| Phase | Duration | Rule |
|---|---|---|
| Development | Open | New strings may be added freely |
| **String freeze** | **7 days before** release | No new strings; existing ones do not change |
| Translation window | Freeze + 7 days | Translators fill in `.po` files |
| Release | Tag push | All `.po` files are compiled with `make compile` |

Urgent bug-fix exception: if a critical user-visible fix requires a new
string, the English fallback is accepted; the translation is completed
in the next patch release.

## Format string sanity

Placeholders (`{name}`, `{n}`) inside `_()` must appear the same number
of times with the same names in every translation. Drift causes a
`KeyError` at runtime.

CI catches this via `msgfmt --check-format`. Local equivalent:

```bash
msgfmt --check --check-format po/tr.po -o /dev/null
```

## Practical maintenance cost

| Operation | Time | Automated? |
|---|---|---|
| `make pot` (per commit) | 1 s | ✅ CI |
| `make update` (new language entries) | 2 s | ✅ CI |
| Translating new strings | ~30 s per string | ❌ human |
| `make compile` (release) | 1 s | ✅ CI |
| Adding a new language (320 strings) | ~4-8 hours | ❌ professional translator |
| Plural-form edge case review | 15 min per release | ❌ |

## File layout

```
po/
├─ Makefile               # build targets
├─ messages.pot           # extracted master template (xgettext)
├─ tr.po                  # Turkish translation (320 entries)
└─ <lang>.po              # additional languages

disk_cleaner/
├─ i18n.py                # gettext wrapper (_, ngettext, reload_translations)
└─ locale/
   └─ tr/LC_MESSAGES/
      └─ disk_cleaner.mo  # compiled binary (read by gettext runtime)
```
